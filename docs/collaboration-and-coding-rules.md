# 协作方式与编码约定（摘要）

> **用途**：给人看的**一页纸摘要**；**对 Cursor 助手的硬约束**以 **`.cursor/rules/*.mdc`**（尤其 **`python-learning-agent.mdc`**）为准。  
> **维护**：当 `.cursor/rules` 中规则变更时，同步检查本摘要是否矛盾。

---

## 1. 助手与代码修改（教学项目约定）

- **默认**：助手**不直接修改** **`*.py`**（含 **`tests/`**）、**`.sh`、`requirements.txt`** 等；以**讲解 + 分步操作说明**为主，由用户在编辑器中完成修改。  
- **`.py` 的唯一例外**：用户在该轮消息正文里出现**一字不差**的连续六字 **`本次允许修改`** 时，助手可在该次任务范围内直接改 `.py`；**「本次允许修改代码」等变体不算**。细则以 **`python-learning-agent.mdc`** 为准。  
- **助手可直接维护的路径**（以规则文件最新版为准）：通常包含 **`.cursor/rules/`**、根目录 **`readme.md`**、以及 **`docs/`** 下与项目说明相关的 **`.md`** 文档。

---

## 2. 对话与教学语言

- **中文**为主；英文术语先给中文释义。  
- **节奏**：可一步一步；用户确认完成后再推进（详见 **`python-learning-agent.mdc`**）。

---

## 3. 编码与工程习惯（摘要）

- **改动范围**：优先最小必要改动，避免无关重构。  
- **风格**：与现有 `src/` 中同目录代码保持一致（命名、分层、日志与错误处理习惯）。  
- **测试**：关键路径变更后运行 **`python -m pytest -q`**（或子集），与 `readme.md` / **`docs/project-goals.md`** 中质量目标一致。  
- **提交前**：更新 **`readme.md`**（当前模块状态与起点）、**`docs/changelog.md`**（本条变更）；**`docs/project-goals.md`** 仅在目标变化时更新。

---

## 4. 权威规则文件索引（`.cursor/rules`）

| 文件 | 作用 |
|------|------|
| **`python-learning-agent.mdc`** | 导师角色、能否改代码、教学节奏、`readme`/`docs` 维护范围 |

| **`project-goal-advanced-agent.mdc`** | 长期高级 Agent 方向 |
